# 1. Overview

This document defines the high-level system architecture for the Adaptive Math Learning platform.

The architecture is designed to support:

- Adaptive mathematics learning
- Knowledge graph remediation
- Mastery tracking
- AMC-style cognitive modeling
- AI-assisted future expansion

The MVP prioritizes:

- Simplicity
- Scalability
- AI-assisted development compatibility
- Clear system boundaries
- Structured data flow

---

# 2. Architecture Philosophy

The platform follows several core architectural principles.

---

## 2.1 AI-Assisted Coding Optimized

The architecture is intentionally structured for:

- Cursor-based development
- AI-generated components
- Modular iteration
- Documentation-first workflows

System boundaries are kept explicit to improve AI context quality.

---

## 2.2 Small but Complete MVP

The MVP emphasizes:

- A fully functional adaptive loop
- Strong ontology structure
- Stable recommendation logic

rather than broad feature coverage.

---

## 2.3 Data-Centric Design

The long-term platform value is expected to come primarily from:

- Knowledge graph quality
- Problem tagging quality
- Learning trajectory data
- Recommendation data

The system is therefore designed around structured educational data.

---

## 2.4 Modular Expansion

The architecture is designed for future support of:

- AP mathematics
- AMC10/12
- AI tutoring
- Problem generation
- Advanced ML systems

without requiring major rewrites.
