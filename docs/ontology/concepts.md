# 1. Overview

This document defines the mathematical concept ontology used in the Adaptive Math Learning system.

The ontology is designed to support:

- Adaptive learning pathways
- Prerequisite reasoning
- Mastery tracking
- Cognitive modeling
- Problem tagging
- Recommendation systems

Unlike traditional curriculum structures, this ontology is:

- Graph-based (not linear)
- Multi-layered (not flat)
- Dependency-driven (not grade-driven)
- Cognitively structured (not textbook-ordered)

---

# 2. Design Principles

## 2.1 Graph-Based Learning Structure

Mathematics is modeled as a directed graph:

Concepts depend on other concepts.

Learning is traversal of this graph.

---

## 2.2 Concept Granularity

Each concept must be:

- Atomic enough to measure mastery
- Broad enough to appear in multiple problems
- Meaningful in isolation

---

## 2.3 Multi-Domain Structure

Concepts span multiple mathematical domains:

- Arithmetic
- Pre-Algebra
- Algebra
- Geometry
- Counting & Probability
- Number Theory

---

## 2.4 AMC-Oriented Modeling

The ontology is optimized for:

- Non-routine problem solving
- Multi-step reasoning
- Pattern recognition
- Concept integration
